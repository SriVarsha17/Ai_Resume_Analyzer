# ATS Grader Backend Service

Node.js, Express, and MongoDB backend supporting authentication, PDF/DOCX file uploading, text extraction, in-memory caching, rate-limiting, and ATS grading utilizing modular diagnostic engines.

---

## 🛠️ Business & Diagnostic Services (`services/`)

The core analytical pipeline is split into separate modules:

1. **`aiService.js`**: Handles generative communications with Gemini Generative AI. Formulates tailored prompts based on target roles, parses structured JSON blocks, and supplies a robust text-scanning fallback if the API fails.
2. **`scoringService.js`**: Normative scoring weights calculations (35% Keyword, 25% Skills, 25% Experience, 15% Format). Estimates shortlist probability percentage based on standard screeners.
3. **`atsAnalyzer.js`**: Scans character streams for formatting elements (tab layouts, non-standard icon headers, pipe symbols, single compressed block walls) and outputs parser risks and fixing tips.
4. **`recruiterSimulationService.js`**: Models human recruiter scanning checks. Computes readability indexes (sentence-to-syllable counts), skimmability balances, visual hotspots, and flags missing metrics or formatting anomalies.
5. **`cacheService.js`**: Map-based caching keyed by SHA-256 hashes of the resume text, job description, and role. Expired entries are garbage-collected every 5 minutes.

---

## 🛡️ Security & Middlewares (`middlewares/`)

- `authMiddleware.js`: Validates JWT Bearer headers and injects user profiles into request contexts.
- `rateLimitMiddleware.js`: Limits requests by IP address (defaults to 60 requests per 15 minutes) to protect against API quota draining.
- `errorMiddleware.js`: Centralized error catching yielding standard JSON stack errors.

## 📂 Models Schema (`models/`)

### `AnalysisResultModel.js`
- `user`: Schema.Types.ObjectId (User reference)
- `resume`: Schema.Types.ObjectId (Resume reference)
- `jobDescriptionText` / `jobDescriptionTitle`: String
- `targetRole`: String
- `atsScore`: Number
- `shortlistChance`: Number
- `breakdown`: `{ keywordScore, skillsScore, experienceScore, formatScore }`
- `sectionScores`: `{ summary, experience, skills, education }`
- `matchedKeywords` / `missingKeywords` / `suggestions`: [String]
- `sectionAnalysis`: `{ skills: { present, details }, experience: { present, details }, education: { present, details } }`
- `bulletPointImprovements`: `[{ original, improved, explanation }]`
- `recruiterSimulation`: `{ skimmabilityScore, readabilityScore, firstImpressionHighlights, redFlags: [{ type, message }] }`
- `atsCompatibility`: `{ riskScore, detectedIssues: [{ element, risk, fix }] }`
- `roleComparison`: `{ missingSkills, experienceGaps, keywordDifferences: [{ word, status }] }`
- `gapRiskDetection`: `{ careerGapsDetected, frequentJobSwitches, suggestions }`

## 📡 API Endpoints

### 📄 Resumes (`/api/resume`)
- `POST /upload` - Uploads a PDF/DOCX file, extracts text, and creates a document (JWT protected).
- `GET /user/all` - Lists all resumes uploaded by the logged-in user (JWT protected).
- `GET /:id` - Fetches a specific resume file metadata and raw text (JWT protected).
- `PUT /:id` - Updates the extracted text of a resume (JWT protected, supports inline editing).

### 📊 Grading & Reports (`/api/analyze`)
- `POST /` - Runs evaluation. Takes `resumeId`, `targetRole`, and either `jdId` or raw pasted text. Returns score report (JWT and Rate Limited).
- `GET /history` - Lists all report histories generated by the user (JWT protected).
- `GET /:id` - Fetches detailed assessment scorecard (JWT protected).
- `DELETE /:id` - Deletes an analysis report (JWT protected).
- `POST /compare` - Takes `analysisId1` and `analysisId2`. Returns delta score, keyword progress, and advanced recruiter simulation gains (JWT protected).
